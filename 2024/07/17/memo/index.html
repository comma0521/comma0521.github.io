cron 表达式和 @every 表达式的详细讲解，以及它们在 cron 包中的使用方法。

cron 表达式

cron 表达式是用来定义时间调度的一种语法，主要由五个字段组成，分别表示分钟、小时、日期、月份和星期几。cron 表达式的格式如下：

每个字段可以包含以下类型的值：

  • *：表示所有合法的值
  • ,：用于分隔多个值
  • -：表示一个范围
  • /：表示步长

示例

  • 0 0 * * *：每天的午夜执行一次
  • 0 9 * * 1：每周一的早上9点执行一次
  • */5 * * * *：每5分钟执行一次
  • 0 0 1 * *：每个月的第一天午夜执行一次

使用示例

@every 表达式

@every 表达式是 cron 包提供的简化调度表达式，用于定义一个周期性的任务调度。它的格式为 @every <duration>，其中 <duration> 是一个合法的 Go 语言时间间隔，比如 1s, 1m, 1h 等。

示例

  • @every 1s：每秒钟执行一次
  • @every 1m：每分钟执行一次
  • @every 1h：每小时执行一次
  • @every 24h：每天执行一次

使用示例

cron 表达式与 @every 表达式的区别

  • cron 表达式：

    • 更加灵活，可以精确地指定在特定的时间点执行任务。
    • 适用于复杂的调度需求，例如每月的某一天或每周的某几天。

  • @every 表达式：

    • 简单易用，用于定义固定周期的任务。
    • 适用于需要定期执行的任务，例如每秒、每分钟、每小时。

选择使用

选择使用哪种表达式取决于你的调度需求：

  • 如果需要在特定的时间点或日期执行任务，使用 cron 表达式。

  • 如果需要定期执行任务，使用 @every 表达式。

通道的概念

通道是一种类型安全的通信机制，允许你在不同的协程之间传递数据。通道通过 make 函数来创建，语法如下：

无缓冲通道和有缓冲通道

无缓冲通道

无缓冲通道要求发送和接收操作必须同步完成。即发送方和接收方都准备好，数据才能在它们之间传递。

有缓冲通道

有缓冲通道允许在没有接收方准备好的情况下，发送方可以发送一定数量的数据。缓冲区的大小由 make 函数的第二个参数指定。
