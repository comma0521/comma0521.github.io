introduction to Linux

what is Unix and Linux

Unix: a family of os

  • was not written in C at first

Linux: a family of Unix-OS

  • free & open source
  • secure
  • multi-user
  • multitasking
  • portability

Linux use cases:

  • Android
  • supercomputers
  • data centers and clout services
  • PCs

Linux distribution(distro): specific flavors of LinuxOS

  • sys utilities

  • GUI

  • shell commands

  • supplort types:

    • LTS
    • community-enterprise

Linux distros:

Debian: stable reliable open source

Ubuntu: debian-based

  • desktop
  • server
  • core(for internet things)

Red Hat Linux: Fedora-based

Fedora:

  • sponsored by RH

SUSE(SLE):

  • SLES: server
  • SLED: developer
  • German

Arch:

  • not stable

Linux Architecture

5 layers:

  • UI:
    • users to interact
  • Application(sys daemons, shells, user apps, tools)
    • sys tools
    • programming languages
  • operating sys:
    • jobs&programs health
  • kernel:
    • vital
    • lowest-level software
    • remain in mem
    • bridge apps and hardware
    • key jobs:
      • mem management
      • process management
      • device drivers
      • security of sys
  • hardware
    • includes
      • CPU
      • RAM
      • Storage
      • Screen
      • USB

Linux filesys

collection of files

tree-like

  • root dirctory:/(root)
    • /bin: binary files(directly below root)
      • codes to run programs and commands
    • /usr: usr program
    • /home: personal working directory
    • /boot: sys boot files
      • for startup
    • /media
    • etc.

Linux terminal overview

Linux shell: an OS-level app that interprets commands

e.g. Bash, Zsh

  • move and copy files
  • wr files
  • extract filter data
  • search for data

Linux terminal: an app you use to interact with the shell

  • enter commands and receive output from them

user enter command to terminal, go to shell, interpret for hardware

and then turn back

terminal command line

  • filepath：

    • ~ home directory
    • / root directory
    • .. parent directory
    • . current directory

  • cd: change directory

    • cd .. go back to parent dir
    • cd ../../ ok

  • ls: list contents of a directory

  • pwd: present working dir

creating and editing txet files

popular text editors:

  • command-line editors:
    • GNU nano
      • nano <filename> create and open a new file
    • vi
    • vim
      • vim <filename>
      • indert mode
        • press i to enter
        • press Esc to exit
        • the text is written to the buffer at the cursor location
      • command mode (default mode)
        • :sav <filename>- create a file and write the buffer to the file
        • :w- write the buffer to the file without exiting
        • :q- quit vim session
        • :q!- quit without saving
        • :wq=:w+:q
  • -based editiors:
    • gedit

cat hello_world.txt- get whatever is in that txt doc.

Installing Software and Updates

packages:

  • archive files
  • for installing new software or updating existing software

package managers:

  • manage the download and installation of packages
  • available for different Linyx distros

Deb and RPM packages

  • for Linux OS
  • .deb files
    • Debian-based distros (e.g. Debian Ubuntu Mint)
    • stands for Debian
  • .rpm files
    • Red Hat -based distros (e.g. CentOS/RHEL, Fedora, openSUSE)
    • stands for Red hat Package Manager
  • both formats are equivalent - can use ailen to convert
    • RPM->deb: alien <package_name>.rpm
    • deb->RPM: alien -r <package_name>.deb

Linux distro package managers

  • PackageKit

    • RPM-based
    • GUI tool

  • yum

    • RPM-based
    • sudo yum update

  • Update Manager

    • deb-based
    • GUI tool
    • automatically check

  • apt

    • deb-based
    • command line
    • sudo apt upgrade

install

  • deb package- sudo apt install<package_name>

  • RPM package- sudo yum install <package_name>

package managers

    • pip
      • pip install pandas
